아래는 **Windows 10 환경에서 `findstr` 명령어를 실무에서 자주 활용하는 대표적인 사용법 5가지**입니다. 각 예시는 실제 운영·개발 환경에서 바로 적용할 수 있도록 설명과 함께 제시합니다.

---

## 1. 특정 문자열이 포함된 줄 검색 (기본 사용법)

```cmd
findstr "ERROR" application.log
```

**설명**

* `application.log` 파일에서 `ERROR` 문자열이 포함된 모든 줄을 출력합니다.
* 대소문자를 구분합니다.

**활용 예**

* 로그 파일에서 에러 메시지 추출
* 배치 작업 실패 원인 파악

---

## 2. 대소문자 구분 없이 검색 (/I)

```cmd
findstr /I "error" application.log
```

**설명**

* `/I` 옵션은 대소문자를 무시하고 검색합니다.
* `ERROR`, `Error`, `error` 모두 검색됩니다.

**활용 예**

* 로그 포맷이 일정하지 않은 경우
* 외부 시스템 로그 분석

---

## 3. 여러 키워드 중 하나라도 포함된 줄 검색

```cmd
findstr "ERROR WARN FAIL" application.log
```

**설명**

* 공백으로 구분된 문자열 중 하나라도 포함되면 출력됩니다.
* OR 조건 검색입니다.

**활용 예**

* 중요 로그 레벨(ERROR/WARN)만 필터링
* 장애 분석 시 주요 키워드 빠르게 확인

---

## 4. 정규식 패턴 검색 (/R)

```cmd
findstr /R "^2025.*ERROR" application.log
```

**설명**

* `/R` 옵션을 사용하면 정규식을 사용할 수 있습니다.
* `^2025` : 2025로 시작하는 줄
* `.*ERROR` : 해당 줄에 ERROR 포함

**활용 예**

* 날짜 기반 로그 분석
* 특정 패턴의 메시지 추출

---

## 5. 특정 문자열이 **없는** 줄 검색 (/V)

```cmd
findstr /V "DEBUG" application.log
```

**설명**

* `/V` 옵션은 조건에 **일치하지 않는 줄**을 출력합니다.
* DEBUG 로그를 제외한 나머지 로그만 표시합니다.

**활용 예**

* 불필요한 로그 제거
* 운영 로그만 선별

---

## 추가로 알아두면 좋은 옵션 요약

| 옵션       | 설명           |
| -------- | ------------ |
| /I       | 대소문자 무시      |
| /R       | 정규식 사용       |
| /V       | 일치하지 않는 줄 출력 |
| /N       | 줄 번호 표시      |
| /C:"문자열" | 정확한 문자열 검색   |

예시:

```cmd
findstr /N /I /C:"connection failed" application.log
```

# 2. 프로세스 관린
아래는 **Windows 10 환경에서 `tasklist` 명령어를 실무에서 자주 사용하는 대표적인 사용법 5가지**입니다. 단순 조회를 넘어 **프로세스 관리·장애 분석·운영 점검** 관점에서 설명합니다.

---

## 1. 현재 실행 중인 모든 프로세스 목록 조회 (기본)

```cmd
tasklist
```

**설명**

* 현재 실행 중인 모든 프로세스를 표시합니다.
* 프로세스 이름, PID, 세션 이름, 메모리 사용량 등을 확인할 수 있습니다.

**활용 예**

* 서버/PC의 전반적인 실행 상태 점검
* 비정상 프로세스 존재 여부 확인

---

## 2. 특정 프로세스 이름으로 필터링 (/FI)

```cmd
tasklist /FI "IMAGENAME eq java.exe"
```

**설명**

* `/FI` 옵션은 조건(Filter)을 적용합니다.
* `java.exe` 프로세스만 조회합니다.

**활용 예**

* Spring Boot, WAS, 배치 프로세스 실행 여부 확인
* Java 기반 서비스 다중 실행 점검

---

## 3. PID 기준으로 특정 프로세스 조회

```cmd
tasklist /FI "PID eq 1234"
```

**설명**

* PID가 1234인 프로세스만 출력합니다.
* 다른 명령어(`netstat -ano` 등)와 연계하여 많이 사용됩니다.

**활용 예**

* 포트 점유 프로세스 추적
* 장애 발생 시 문제 프로세스 식별

---

## 4. 메모리 사용량 기준으로 필터링

```cmd
tasklist /FI "MEMUSAGE gt 500000"
```

**설명**

* 메모리 사용량이 500,000KB(약 500MB) 초과인 프로세스만 출력합니다.
* 조건식: `gt`, `lt`, `eq`

**활용 예**

* 메모리 누수 의심 프로세스 탐지
* 리소스 사용 과다 프로세스 점검

---

## 5. 서비스 정보 포함하여 조회 (/SVC)

```cmd
tasklist /SVC
```

**설명**

* 각 프로세스에 연결된 Windows 서비스 목록을 함께 표시합니다.
* `svchost.exe` 내부 서비스 식별에 매우 유용합니다.

**활용 예**

* 특정 서비스가 어느 프로세스에서 실행 중인지 확인
* 서비스 장애 원인 분석

---

## 실무에서 자주 쓰는 추가 조합 예시

### ▸ 프로세스 + 서비스 + 필터링

```cmd
tasklist /SVC /FI "IMAGENAME eq svchost.exe"
```

### ▸ 원격 서버 프로세스 조회

```cmd
tasklist /S remote_server /U admin /P password
```

---

## 주요 필터 조건 정리

| 조건        | 설명          |
| --------- | ----------- |
| IMAGENAME | 실행 파일 이름    |
| PID       | 프로세스 ID     |
| MEMUSAGE  | 메모리 사용량(KB) |
| STATUS    | 실행 상태       |
| USERNAME  | 실행 사용자      |

---

## 참고: `tasklist` + `taskkill` 연계

```cmd
taskkill /PID 1234 /F
```

* `tasklist`로 확인 → `taskkill`로 종료
* 운영 환경에서는 반드시 PID 확인 후 사용 권장

# 정규식
아래는 **실무 및 교육 현장에서 가장 빈번하게 사용되는 자바 정규식(Regex) 예제 5가지**입니다. 각 항목마다 **정규식 패턴의 의미**와 **Java 코드 예시**를 함께 제시합니다.

---

## 1. 이메일 주소 검증

### 정규식

```regex
^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+$
```

### 설명

* 일반적인 이메일 형식 검증
* 회원가입, 사용자 정보 입력 시 가장 많이 사용

### Java 예제

```java
String email = "user@example.com";
boolean isValid = email.matches("^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+$");
```

---

## 2. 휴대폰 번호 검증 (대한민국 기준)

### 정규식

```regex
^01[016789]-?\d{3,4}-?\d{4}$
```

### 설명

* `010-1234-5678`, `01112345678` 모두 허용
* 하이픈(-) 포함 여부 선택적

### Java 예제

```java
String phone = "010-1234-5678";
boolean isValid = phone.matches("^01[016789]-?\\d{3,4}-?\\d{4}$");
```

---

## 3. 숫자만 포함되어 있는지 검사

### 정규식

```regex
^\d+$
```

### 설명

* 입력값이 **순수 숫자**인지 확인
* 나이, 수량, 금액 입력 검증에 활용

### Java 예제

```java
String input = "12345";
boolean isNumber = input.matches("^\\d+$");
```

---

## 4. 비밀번호 정책 검증 (영문 + 숫자 + 특수문자, 8자 이상)

### 정규식

```regex
^(?=.*[A-Za-z])(?=.*\d)(?=.*[!@#$%^&*()_+]).{8,}$
```

### 설명

* 영문자 1개 이상
* 숫자 1개 이상
* 특수문자 1개 이상
* 최소 8자 이상

### Java 예제

```java
String password = "Abc123!@";
boolean isValid = password.matches(
    "^(?=.*[A-Za-z])(?=.*\\d)(?=.*[!@#$%^&*()_+]).{8,}$"
);
```

---

## 5. 공백 제거 또는 공백 포함 여부 검사

### 정규식

```regex
\s+
```

### 설명

* 문자열 내 모든 공백 검색
* 문자열 정제(sanitize) 작업에서 자주 사용

### Java 예제 (공백 제거)

```java
String text = "Java   Regex  Test";
String result = text.replaceAll("\\s+", "");
// 결과: JavaRegexTest
```

---

### 정리 요약

| 용도      | 정규식                                         |
| ------- | ------------------------------------------- |
| 이메일 검증  | `^[A-Za-z0-9+_.-]+@[A-Za-z0-9.-]+$`         |
| 휴대폰 번호  | `^01[016789]-?\d{3,4}-?\d{4}$`              |
| 숫자 여부   | `^\d+$`                                     |
| 비밀번호 정책 | `^(?=.*[A-Za-z])(?=.*\d)(?=.*[특수문자]).{8,}$` |
| 공백 처리   | `\s+`                                       |

---

아래는 **입력값이 숫자인지, 문자인지 확인할 때 가장 많이 사용하는 자바 정규식 예제**를 목적별로 정리한 것입니다. 실무에서 바로 적용 가능한 형태로 설명합니다.

---

## 1. 숫자만 입력되었는지 확인

### 정규식

```regex
^\d+$
```

### 의미

* `^` : 문자열 시작
* `\d+` : 숫자(0~9) 1개 이상
* `$` : 문자열 끝

### Java 예제

```java
String input = "123456";
boolean isOnlyNumber = input.matches("^\\d+$");
```

---

## 2. 문자(영문자)만 입력되었는지 확인

### 정규식

```regex
^[A-Za-z]+$
```

### 의미

* 대문자 A~Z, 소문자 a~z만 허용
* 숫자, 특수문자, 공백 불허

### Java 예제

```java
String input = "JavaRegex";
boolean isOnlyAlphabet = input.matches("^[A-Za-z]+$");
```

---

## 3. 한글만 입력되었는지 확인

### 정규식

```regex
^[가-힣]+$
```

### 의미

* 완성형 한글만 허용
* 자음·모음 단독 입력은 불가

### Java 예제

```java
String input = "홍길동";
boolean isOnlyKorean = input.matches("^[가-힣]+$");
```

---

## 4. 숫자 또는 문자(영문)만 입력되었는지 확인

### 정규식

```regex
^[A-Za-z0-9]+$
```

### 의미

* 영문자 + 숫자 조합 허용
* 특수문자, 공백 불허

### Java 예제

```java
String input = "Java123";
boolean isAlphaNumeric = input.matches("^[A-Za-z0-9]+$");
```

---

## 5. 숫자 또는 한글만 입력되었는지 확인

### 정규식

```regex
^[0-9가-힣]+$
```

### Java 예제

```java
String input = "서울123";
boolean isValid = input.matches("^[0-9가-힣]+$");
```

---

## 6. 문자 포함 여부만 확인 (숫자 혼합 가능)

### 정규식

```regex
.*[A-Za-z].*
```

### 의미

* 문자열 내에 **문자가 하나라도 포함**되어 있으면 통과

### Java 예제

```java
String input = "123A45";
boolean containsAlphabet = input.matches(".*[A-Za-z].*");
```

---

## 7. 숫자 포함 여부만 확인 (문자 혼합 가능)

### 정규식

```regex
.*\d.*
```

### Java 예제

```java
String input = "Test1";
boolean containsNumber = input.matches(".*\\d.*");
```

---

### 핵심 정리 표

| 목적       | 정규식              |
| -------- | ---------------- |
| 숫자만      | `^\d+$`          |
| 영문자만     | `^[A-Za-z]+$`    |
| 한글만      | `^[가-힣]+$`       |
| 영문+숫자    | `^[A-Za-z0-9]+$` |
| 한글+숫자    | `^[0-9가-힣]+$`    |
| 문자 포함 여부 | `.*[A-Za-z].*`   |
| 숫자 포함 여부 | `.*\d.*`         |
