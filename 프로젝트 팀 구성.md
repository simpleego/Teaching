# 프로젝트 팀 구성
비전공자로서 4개월 과정 후 LMS 프로젝트 팀 구성은 매우 중요한 결정입니다. 다음 기준으로 팀원을 선정하는 것을 추천드립니다:

## 👥 **팀 구성 핵심 기준**

### **1. 기술적 능력 균형**

```markdown
**프론트엔드 집중**
- HTML/CSS/JavaScript 숙련도
- React/Vue 등 프레임워크 이해도
- 반응형 디자인 구현 능력

**백엔드 집중**
- Node.js/Express 또는 Spring 이해도
- 데이터베이스 설계 능력 (MySQL, MongoDB)
- API 설계 및 구현 경험

**풀스택 가능성**
- 프론트/백엔드 모두 기본 이해
- 전체 아키텍처 이해력
```

### **2. 협업 능력 평가**

```markdown
**의사소통 스킬**
- □ 의견 표현이 명확한가?
- □ 피드백 수용 능력
- □ 적극적인 참여도

**문제 해결 태도**
- □ 어려움에 직면했을 때의 태도
- □ 스스로 해결하려는 의지
- □ 도움 요청하는 방법 아는가?

**시간 관리**
- □ 데드라인 인식
- □ 일정 계획성
- □ 약속 준수도
```

## 🔍 **팀원 선정 실전 가이드**

### **1. 기술 스택 매핑**
```
[LMS 필수 기능별 필요 기술]
✅ 회원가입/로그인 - 백엔드 (세션, JWT)
✅ 강의 목록/상세 - 프론트엔드 (UI/라우팅)
✅ 수강 신청/결제 - 풀스택 (프론트+백 연동)
✅ 관리자 페이지 - 백엔드 (CRUD 구현)
✅ 반응형 디자인 - 프론트엔드 (CSS)
```

### **2. 협업 스타일 확인 질문지**

```python
# 팀원과 공유할 질문들
```

## ⚠️ **피해야 할 팀 구성**

### **위험 신호**
```markdown
❌ **기술 편중 팀**: 프론트엔드만 많거나 백엔드만 많은 팀
❌ **수동적 팀원**: 기다리기만 하는 사람
❌ **고립형**: 소통하지 않는 사람
❌ **완벽주의자**: 너무 깐깐해서 진행이 느린 사람
❌ **의존적**: 모든 결정을 팀에 미루는 사람
```

## 🎯 **비전공자 특화 전략**

### **강점 활용**

```javascript
const nonMajorStrengths = {

```

### **학습 계획 수립**
```markdown
**1주차: 기초 공통 학습**
- 팀 전체가 기본 개념 함께 복습
- Git 협업 workflow 설정

**2-3주차: 역할별 심화**
- 각자 담당 파트 집중 학습
- 데일리 스크럼으로 진행상황 공유

**4주차 이후: 통합 및 개선**
- 기능 통합 테스트
- 코드 리뷰 및 리팩토링
```

## 📝 **실제 팀 구성 예시**

### **Case 1: 안정적인 팀**
```
👤 홍길동: 풀스택 (팀장) - 전체 아키텍처 담당
👤 김철수: 프론트엔드 - UI 컴포넌트 개발  
👤 이영희: 백엔드 - API 및 데이터베이스
👤 박지민: 프론트엔드 - 페이지 라우팅 및 상태관리
```

### **Case 2: 도전적인 팀**
```
👤 정소미: 프론트엔드 - 디자인 시스템
👤 최윤호: 백엔드 - 인증/보안 담당
👤 한동훈: 풀스택 - 주요 기능 개발
👤 김수진: 테스트/QA - 품질 관리
```

## 💡 **성공적인 팀운영을 위한 Tip**

```markdown
1. **주간 회의 필수**: 매주 목표 설정과 회고
2. **코드 컨벤션 통일**: 코딩 스타일 미리 합의
3. **Git 전략 수립**: branch 전략, commit 규칙
4. **커뮤니케이션 채널**: Slack/Discord 등 실시간 소통
5. **어려움 공유 문화**: 막히면 바로 도움 요청
```
