# 네트워크 이론 및 실습
> 네트워크 프로그래밍은 단순히 코드를 작성하는 것을 넘어, 데이터가 물리적/논리적 통로를 어떻게 통과하는지 이해하는 과정입니다.
>  **5일간(일일 8시간)** 집중적으로 파이썬 네트워크 프로그래밍을 마스터할 수 있는 커리큘럼

---

## 🗓️ 1일차: 네트워크 기초 및 소켓 프로그래밍 입문

**목표:** OSI 7계층 이해와 기본적인 TCP 통신 구현

* **오전 (4시간): 이론 및 환경 설정**
* 네트워크 계층 구조 (OSI 7 Layer vs TCP/IP 4 Layer).
* IP 주소, 포트(Port), 프로토콜(TCP/UDP)의 역할.
* **Socket**의 개념: 엔드포인트 간의 연결 통로.


* **오후 (4시간): 실습**
* 파이썬 `socket` 모듈 기초 API 학습 (`socket()`, `bind()`, `listen()`, `accept()`).
* **Echo Server/Client** 구현: 클라이언트가 보낸 메시지를 서버가 그대로 돌려주는 기본 모델.
* `struct` 모듈을 이용한 바이너리 데이터 패킹/언패킹 실습.



---

## 🗓️ 2일차: TCP와 UDP Deep Dive

**목표:** 연결 지향형(TCP)과 비연결형(UDP) 통신의 차이 및 활용

* **오전 (4시간): TCP 심화**
* TCP 3-way Handshake 및 4-way Teardown 절차.
* 데이터 유실 방지를 위한 재전송 메커니즘 이해.
* 실습: 대용량 파일 전송 프로그램 작성 (Chunk 단위 전송).


* **오후 (4시간): UDP 프로그래밍**
* UDP의 특성: 비연결성, 빠른 속도, 데이터 순서 불일치.
* 실습: UDP를 이용한 간단한 채팅 프로그램 또는 실시간 시간 브로드캐스트.
* **Wireshark**를 사용해 실제 패킷이 어떻게 오가는지 분석하기.



---

## 🗓️ 3일차: 다중 접속 처리 (Concurrency)

**목표:** 여러 클라이언트가 동시에 접속할 수 있는 서버 구축

* **오전 (4시간): 멀티프로세싱과 멀티스레딩**
* `threading`과 `multiprocessing` 모듈의 차이와 선택 기준.
* Race Condition 방지를 위한 Lock/Semaphore 기초.
* 실습: Threaded Echo Server 구현.


* **오후 (4시간): 비동기 I/O (Asyncio)**
* Blocking vs Non-blocking I/O, Synchronous vs Asynchronous.
* 파이썬 `asyncio` 모듈과 `await`, `async` 키워드 숙달.
* 실습: `asyncio`를 활용한 고성능 동시성 서버 구현.



---

## 🗓️ 4일차: 상위 계층 프로토콜과 HTTP

**목표:** 웹의 근간인 HTTP와 보안(TLS/SSL) 적용

* **오전 (4시간): HTTP 및 API 통신**
* HTTP 메서드(GET, POST, PUT, DELETE)와 상태 코드.
* `requests` 라이브러리를 활용한 REST API 클라이언트 개발.
* `urllib`를 이용한 로우 레벨 URL 핸들링.


* **오후 (4시간): 보안 및 데이터 포맷**
* JSON, XML 데이터 파싱 및 전송.
* SSL/TLS의 원리: 공개키 암호화 방식 이해.
* 실습: `ssl` 모듈을 적용한 암호화된 TCP 서버/클라이언트 구축.



---

## 🗓️ 5일차: 고급 라이브러리 및 프로젝트 실습

**목표:** 실제 상용 수준의 라이브러리 활용 및 종합 프로젝트

* **오전 (4시간): 고급 프레임워크**
* **Scapy**: 패킷 조작 및 네트워크 분석 도구 활용.
* **FastAPI** 또는 **Flask**: 네트워크 소켓 지식을 바탕으로 한 웹 서버 구축 맛보기.


* **오후 (4시간): 종합 프로젝트**
* **주제:** 멀티룸 지원 실시간 채팅 서버 또는 파일 동기화 프로그램.
* 설계 -> 구현 -> 디버깅 -> 최적화 과정 수행.
* 예외 처리(Connection Reset, Timeout 등)를 포함한 견고한 코드 작성.
