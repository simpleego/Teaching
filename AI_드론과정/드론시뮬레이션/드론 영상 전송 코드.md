


## 드론 영상 전송 코드
- drone.start_video_stream(duration=10, show_window=True)
> 이 코드는 드론을 제어하는 라이브러리(일반적으로 `DJITelloPy`나 유사한 드론 제어 SDK)를 사용하여 **드론의 카메라 영상을 실시간으로 송출하고 화면에 표시**하도록 명령하는 코드입니다.

각 매개변수의 역할과 전체적인 흐름을 알기 쉽게 정리

---

### ## 코드 구성 요소 분석

`drone.start_video_stream(duration=10, show_window=True)`

* **`drone`**: 드론 객체입니다. 이전에 `drone = Tello()`와 같은 방식으로 연결된 드론 본체를 가리킵니다.
* **`.start_video_stream()`**: 드론의 카메라를 활성화하여 비디오 데이터 전송을 시작하라는 메서드(함수)입니다.
* **`duration=10`**: 비디오 스트림을 유지할 **지속 시간**을 설정합니다. 여기서는 **10초** 동안 영상을 받겠다는 의미입니다.
* **`show_window=True`**: 컴퓨터 화면에 **별도의 팝업 창**을 띄워 드론이 촬영 중인 화면을 실시간으로 보여줄지 여부를 결정합니다. `True`이므로 화면에 영상 창이 나타납니다.

---

### ### 코드 실행 흐름

1. **스트림 요청**: 컴퓨터가 드론에게 "카메라 화면을 보내줘"라고 요청합니다.
2. **데이터 전송**: 드론이 Wi-Fi를 통해 UDP 패킷 형태로 영상 데이터를 컴퓨터에 보냅니다.
3. **디코딩 및 출력**: 컴퓨터가 받은 데이터를 우리가 볼 수 있는 화면으로 변환(Decoding)하여 창에 띄웁니다.
4. **종료**: 10초가 지나면 자동으로 영상 송출을 중단하고 창을 닫습니다.

---

### 💡 참고하세요!

* **네트워크 상태**: 드론과 컴퓨터 사이의 Wi-Fi 연결이 불안정하면 영상에 지연(Latency)이 생기거나 끊길 수 있습니다.
* **배터리 소모**: 영상 스트리밍은 드론의 배터리를 꽤 많이 소모하므로, 필요할 때만 켜는 것이 좋습니다.
* **라이브러리 확인**: 사용하는 라이브러리에 따라 `duration` 대신 `stop_video_stream()`을 직접 호출해야 하는 경우도 있으니 매뉴얼을 확인해 보세요.
