# ✅ **예시 1 — 학생 A**

## 1) 사전면담결과

* 프로젝트에 대한 참여 의지가 높고, 일정 준수에 대한 책임감이 강함.
* 팀 내 갈등 발생 시 중재 역할을 선호하며, 의견 조율에 능숙한 성향을 보임.
* 이전 소규모 개인 프로젝트 경험을 통해 기본적인 Git 사용 경험이 있고, 협업 도구 활용에도 긍정적인 태도를 보임.

## 2) 학생 역량분석

* Spring Boot 기반의 REST API 구성 및 MyBatis CRUD 구현 능력이 안정적임.
* 오류 분석 및 해결 속도가 빠르며 백엔드 로직 이해도가 높음.
* 데이터베이스 스키마 설계 실습 과정에서 높은 평가를 받아 백엔드/DB 파트에 강점이 있음.
* 기술 문서 작성 능력이 좋아 API 명세서 작성 등에서도 기여 가능.

## 3) 선호분야 조사 결과

* **희망 역할:** 백엔드 개발자
* **선호 기술:** Spring, MyBatis, DB 설계
* **기피 분야:** UI 스타일링, 디자인
* **선호 이유:** 실무 백엔드 개발자로 취업 희망
  → **배치 근거:** 백엔드 중심 팀에 배치하여 서버 개발 및 DB 설계를 담당하도록 적합

# ✅ **예시 2 — 학생 B**

## 1) 사전면담결과

* 팀 활동에 적극적으로 참여하며, 프론트엔드에 대한 흥미가 높아 새로운 라이브러리 학습에도 적극적임.
* 협업 시 의견을 명확히 전달하는 스타일이며 발표나 데모 진행에 부담이 없음.
* 시간 배분이 다소 느슨한 경향이 있어 일정 관리 및 기능 분배 시 명확한 기준 제시가 필요함.

## 2) 학생 역량분석

* React 컴포넌트 구조 이해도가 높고, 상태 관리(useState, useEffect, useContext 등) 사용에 익숙함.
* UI 구성 능력이 뛰어나며 시각적 표현 능력이 강점.
* 프론트엔드 빌드/배포 경험은 부족하지만 학습 속도가 빠름.
* Git 브랜치 전략 숙련도가 부족해 추가적인 가이드 필요.

## 3) 선호분야 조사 결과

* **희망 역할:** 프론트엔드(UI 개발)
* **선호 기술:** React, Tailwind CSS
* **기피 분야:** 서버 API 작성, DB 작업
* **선호 이유:** 시각 기반 UI 개발에 흥미
  → **배치 근거:** 프론트엔드 화면 구성 및 사용자 경험(UX) 개선 담당으로 적합

# ✅ **예시 3 — 학생 C**

## 1) 사전면담결과

* 조용하지만 성실한 업무 수행 스타일이며 주어진 역할을 책임감 있게 완수하는 유형.
* 팀 내 리더 역할보다는 일정한 개발 업무를 꾸준히 맡는 것을 선호.
* 문서 작성, 요구사항 정리 등 기획 관련 작업에 관심을 보임.

## 2) 학생 역량분석

* 요구사항 분석 및 기능 정의를 구조화하는 능력이 뛰어남.
* ERD 작성 및 업무 흐름도 설계 능력이 안정적이며 테스트 과정에서 꼼꼼함을 보임.
* 개발 속도는 빠르지 않지만 정확성이 높으며 QA 과정에서 오류를 잘 잡아냄.
* 기획 문서, 기능 명세서 작성에 강점.

## 3) 선호분야 조사 결과

* **희망 역할:** 기획 및 QA, 간단한 프론트 보조
* **선호 기술:** Figma, Notion, React 기초
* **기피 분야:** 복잡한 서버 및 DB 개발
* **선호 이유:** 구조화·문서화 업무 선호
  → **배치 근거:** 기획·문서 작성 및 QA 중심 역할로 팀 안정성 향상에 기여 가능
